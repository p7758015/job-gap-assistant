# Job Gap Assistant (AI Career Analyzer)

`job-gap-assistant` — AI-сервис для анализа соответствия между требованиями вакансии и текущими навыками кандидата.

Проект демонстрирует, как с помощью LLM можно автоматически:

* разбирать вакансии,
* оценивать уровень соответствия кандидата,
* выявлять пробелы в навыках,
* формировать персональный план развития.

⚡ **Важно:** проект полностью реализован в формате *vibe coding* с использованием **Cursor (AI-IDE)** — архитектура, API и логика были сгенерированы и доработаны с помощью AI-ассистента.

---

## Что делает сервис

Сервис принимает:

* `job_description` — текст вакансии
* `candidate_profile` — описание текущего опыта кандидата (опционально)

И возвращает структурированный результат:

* ключевые навыки (must-have / nice-to-have / bonus)
* `fit_score` (оценка соответствия 0–100%)
* объяснение оценки
* список пробелов (gaps)
* проекты для прокачки
* roadmap развития на 2–4 недели

## Быстрый старт (локально)

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-repo/job-gap-assistant.git
cd job-gap-assistant
```

### 2. Установка зависимостей

С использованием `uv`:

```bash
uv pip install -e .
```

или через `pip`:

```bash
pip install -e .
```

### 3. Переменные окружения

Скопируйте `.env.example` в `.env` и укажите ключ OpenAI:

```bash
OPENAI_API_KEY=your_key_here
```

Опционально:

```bash
OPENAI_MODEL=gpt-4.1-mini
```

Файл `.env` добавлен в `.gitignore` и не должен попадать в репозиторий.

### 4. Запуск

```bash
uvicorn app.main:app --reload --port 8000
```

После запуска:

* Health-check: http://127.0.0.1:8000/health
* Swagger UI: http://127.0.0.1:8000/docs

## API

### Анализ вакансии и профиля

**POST `/analyze`**

Пример запроса:

```json
{
  "job_description": "We are looking for a Python backend engineer with FastAPI and PostgreSQL experience.",
  "candidate_profile": "I have 2 years of experience with Python and FastAPI, basic PostgreSQL."
}
```

Пример ответа:

```json
{
  "skills": {
    "must_have": ["FastAPI", "PostgreSQL", "Python"],
    "nice_to_have": ["Testing"],
    "bonus": ["System design basics"]
  },
  "fit_score": 78,
  "fit_explanation": "Ваш профиль покрывает ключевые требования...",
  "gaps": ["Deep PostgreSQL integration"],
  "projects": [
    {
      "title": "FastAPI backend с PostgreSQL",
      "description": "Реализовать API с БД и тестами",
      "estimated_duration_weeks": 2
    }
  ],
  "roadmap": [
    {
      "week": 1,
      "focus": "Тестирование API",
      "tasks": ["Покрыть endpoints тестами"]
    }
  ]
}
```

Ответ строго валидируется через Pydantic-схему `AnalyzeResponse`.

## Архитектура и подход

Проект демонстрирует современный подход к разработке AI-сервисов:

## Limitations

* качество анализа зависит от LLM
* возможны неточности (hallucinations)
* результат стоит использовать как рекомендацию, а не финальное решение
